# 代码修改说明 - K230连续数据模式

## 修改内容

### 1. JSON解析函数重写
**位置**：`User/main.c` 第303-375行

**修改前**：期望一次性接收所有4个位置的数据
**修改后**：支持K230的单物品连续发送模式

**新的JSON格式支持**：
```json
{"cy": 233, "confidence": 0.36, "position": "bottom_right", "label": "egg", "cx": 534}
```

### 2. 数据处理逻辑优化
**位置**：`User/main.c` 第387-425行

**修改内容**：
- 每次收到K230数据立即解析
- 如果有称重数据就立即合并
- 如果没有称重数据就发送物品识别信息

### 3. 清理重复代码
删除了旧代码残留，确保函数完整性

## 工作流程

### 数据流程
```
K230连续发送 → CH32V307接收 → 解析单个物品数据 → 
                                         ↓
                                有称重数据？
                                    ↓
                              是 → 立即合并 → 发送到PC
                              否 → 缓存物品信息 → 等待称重
```

### 通道映射
```
top_left     → 通道0 (ch1)
top_right    → 通道1 (ch2)
bottom_left  → 通道2 (ch3)
bottom_right → 通道3 (ch4)
```

## 故障排除

### 问题1：称重一直失败
**检查**：
- ADS1234硬件连接
- DOUT引脚电平
- 通道切换是否正常

### 问题2：JSON解析失败
**检查**：
- K230发送的格式是否正确
- position字段值是否匹配（top_left等）
- label字段是否存在

### 问题3：数据不合并
**检查**：
- g_weights_updated和g_items_updated的状态
- 两个数据源是否都正常工作
