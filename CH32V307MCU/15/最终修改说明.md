## 🎯 关键修改

### 1. JSON解析适配K230格式

**K230实际发送**：
```json
{"cy": 31, "confidence": 0.26, "position": "top_right", "label": "egg", "cx": 501}
```

**新解析逻辑**：
- 读取 `position` 字段 → 确定通道（0-3）
- 读取 `label` 字段 → 物品名称
- 读取 `confidence` 字段 → 置信度

### 2. 位置与通道映射

```
K230位置        →  通道   →  说明
"top_left"     →  通道0   →  ch1  左上
"top_right"    →  通道1   →  ch2  右上
"bottom_left"  →  通道2   →  ch3  左下
"bottom_right" →  通道3   →  ch4  右下
```

### 3. 调试信息增强

现在会输出详细的调试信息到UART1（调试串口）：
```
[JSON] 解析K230数据: {...}
[JSON] position=top_right
[JSON] label=egg
[JSON] confidence=0.26
[JSON] 解析成功: 通道1(top_right) = egg, 置信度=0.26
```

## ⚠️ 重要提示

### 关于ADC=0的问题
当前代码**暂时允许ADC=0通过**，用于测试数据流程。

**如果ADC一直返回0，可能是**：
1. ADS1234硬件连接问题
2. DOUT引脚一直是高电平
3. 通道切换失败
4. 芯片未正常初始化

### 如何检查硬件
1. 用万用表检查DOUT（PA1）电平
2. 检查SCLK（PA0）是否有脉冲输出
3. 检查A0（PB2）和A1（PB5）电平
4. 确认PDWN（PB0）是高电平
